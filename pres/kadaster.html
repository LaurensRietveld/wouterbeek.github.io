How to Query Cadastral Big Data?

Or: How to SPARQL 100,000,000,000 statements without blowing up?

Properties of Cadastral Big Data

  • Billions of statements
  • Worst-case precision at least 1.0e-6 (~11cm)
  • High ánd low complexity geometries
  • 2D ánd 3D geometries: projection currently not standardized
  • Fast data joins
  • Fast filters
  • Fast geo functions
  • Fast schema queries

Stakeholder Wishlist

  • External ánd internal rings: the famous ‘donut’ problem.
  • Syntactic sugar for mapping functions → queries (e.g., geo:sfWithin)
  • Accept multiple serializations: WKT, GML

SELECT ?country
PROJECTION {
  "capital": ?capital,
  "provinces": [?province],
  "population": ?population
} WHERE {
  ?country def:hasCapital ?capital ;
           def:hasProvince ?province ;
           def:hasPopulation ?population .
