How is syntax killing the SW?

  • Too many formats with too little differentiation
  • Inability to represent N-ary predicates concisely
  • Literal equivalence test requires lexical-to-value mapping due to non-canonical lexical expressions
  • IRI equivalence test is cumbersome due to excessive use of chracter encoding
  • Blank node identifiers are not forced to be universally unique

The (bad) syntax obsession

The SW community believes that the lack of SW uptake is mainly due to others using the wrong syntax:

  • "SW failed due to RDF/XML"
  • "JSON-LD will save the SW!"
  • "We used to be bogged down in XML and XSLT but now we have JSON-LD and JSON-LD Fragments!"
  • "We need to update the syllabus for the SW course to use Turtle i.o. XML/RDF. Students will love it!"
JSON-LD as a Recommendation has helped lower the defenses of many who used to get their hackles up and say 'RDF is too hard'.

The (bad) syntax obsession

SW researcher: "I build this really cool algorithm that performs querying|reasoning quite well."

RDF syntax according to Wouter Beek

The difference between RDF/XML and JSON-LD according to Wouter Beek

<rdf:RDF>
...
</rdf:RDF>

}

The similarity:

  • Tree/graph: tree-based formats that need arcane constructs to represent graph data
  • Streaming 1: required start and end tags/symbols make sure most substreams of the datastream are ill-formed
  • Streaming 2: prefix declarations make sure all substreams without the header are ill-formed.

RDF syntax according to Wouter Beek

  • Don't use tree-based format for graph-data (exit RDF/XML|JSON-LD).
  • End-of-triple should be one unique character (exit Turtle).
  • Allow RDF statement to be split into its 3|4 RDF terms with a simple RE.
  • Every substream must be fully interpretable except for (possibly) the first and last statement.
  • Named graphs must be supported.

The existing format that scores best on these criteria: Canonical N-Quads.

Advice Pleas from a SW PhD

  • Do not spend the next 15 years on 8 new formats.
  • Talk about semantics and pragmatics and only talk about syntax to the extend that it impacts talk about semantics and pragmatics.
  • Use software tools in order to look at data.
  • Do not consider raw data files for bedtime reading.
  • Publish your data in only one syntactic format but do it well.
  • Use the LOD Laundromat to do this for you.
  • Spend the time you save on jugling syntax with your family or by reading poetry.
